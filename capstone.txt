Product Management API System
1. Project Overview
In this capstone project, will design and implement a complete backend API system that manages:
•	Users
•	Products
•	Categories
The system will expose a RESTful API that allows clients (frontend apps to interact with the backend.
2. System Modules
The system will include three main entities.
2.1 Users
Represents users who can access the system.
Example fields:
id
name
email
password
role
createdAt
Roles may include:
admin
user
________________________________________
2.2 Categories
Represents product categories.
Example fields:
id
name
description
createdAt
________________________________________
2.3 Products
Represents items sold or managed in the system.
Example fields:
id
name
description
price
image
categoryId
createdBy
createdAt
Relationship example:
Category (1) -------- (Many) Products
User (1) -------- (Many) Products
